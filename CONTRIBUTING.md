Thank you for contributing! Please follow these simple steps to collaborate.

Branching
- Work on feature branches named `feature/<short-description>` or `fix/<short-description>`.
- Push branches to the remote and open a Pull Request targeting `develop`.

Pull Requests
- Open PRs against `develop` with a clear title and description.
- Link related issues and provide testing steps.
- Keep PRs small and focused; prefer one logical change per PR.

Commits
- Use meaningful commit messages. Prefer present-tense summary on the first line.

Code review
- Request at least one reviewer. Address review comments via new commits.

Release flow
- Merges to `develop` are reviewed; create release PRs from `develop` into `master` when ready.

Thanks — maintainers
